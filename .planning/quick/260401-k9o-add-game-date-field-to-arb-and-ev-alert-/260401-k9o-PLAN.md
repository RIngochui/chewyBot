
Note on strftime: `%-d` and `%-I` use Linux/macOS format codes to strip leading zeros (e.g. "Apr 5" not "Apr 05", "7:30 PM" not "07:30 PM"). This is correct for the target host environment (Linux server). The datetime from the API is UTC — `.astimezone()` on an aware datetime handles the UTC→ET conversion correctly. If start_time is naive UTC, call `.replace(tzinfo=ZoneInfo("UTC")).astimezone(ZoneInfo("America/New_York"))` instead.

Check models/odds.py to confirm whether `start_time` is timezone-aware or naive before implementing. If naive, use the replace() approach. If aware (tzinfo set), use .astimezone() directly.
