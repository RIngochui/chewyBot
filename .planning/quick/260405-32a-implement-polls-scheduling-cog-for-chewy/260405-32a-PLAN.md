DDL — three new tables to append to CREATE_TABLES_SQL:

```sql
-- Active and completed poll instances
CREATE TABLE IF NOT EXISTS polls (
    id                  INTEGER PRIMARY KEY AUTOINCREMENT,
    question            TEXT NOT NULL,
    options             TEXT NOT NULL,          -- JSON array of option strings (max 9)
    channel_id          INTEGER NOT NULL,
    discord_message_id  TEXT,
    max_choices         INTEGER,                -- NULL = unlimited
    post_at             TIMESTAMP,              -- NULL = posted immediately
    close_at            TIMESTAMP NOT NULL,
    closed              INTEGER DEFAULT 0,
    recurring_poll_id   INTEGER REFERENCES recurring_polls(id),
    created_at          TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)

-- Recurring weekly poll definitions
CREATE TABLE IF NOT EXISTS recurring_polls (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    question    TEXT NOT NULL,
    options     TEXT NOT NULL,                  -- JSON array
    channel_id  INTEGER NOT NULL,
    max_choices INTEGER,                        -- NULL = unlimited
    day_of_week TEXT NOT NULL,                  -- e.g. "Monday"
    post_time   TEXT NOT NULL,                  -- e.g. "09:00"
    close_time  TEXT NOT NULL,                  -- e.g. "18:00"
    active      INTEGER DEFAULT 1,
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)

-- Per-user vote tracking (one row per user per option per poll)
CREATE TABLE IF NOT EXISTS poll_votes (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    poll_id     INTEGER NOT NULL REFERENCES polls(id),
    user_id     TEXT NOT NULL,
    option_idx  INTEGER NOT NULL,               -- 0-based index into options array
    voted_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(poll_id, user_id, option_idx)
)
```

NOTE: SQLite evaluates REFERENCES lazily; the forward reference `recurring_polls(id)` in polls DDL is fine if recurring_polls is created first. Reorder the three entries so recurring_polls comes before polls and poll_votes in CREATE_TABLES_SQL.

HELPER: `_parse_datetime(s: str) -> datetime | None`
Try parsing common formats:
1. `dateparser.parse(s, settings={"RETURN_AS_TIMEZONE_AWARE": True, "PREFER_DATES_FROM": "future"})` if dateparser available
2. Manual fallback:
   - ISO "YYYY-MM-DD HH:MM" → `datetime.strptime(s, "%Y-%m-%d %H:%M").replace(tzinfo=timezone.utc)`
   - "Weekday H:MMam" or "Weekday H:MMpm" → compute next occurrence of that weekday at that time (UTC)
Return None if unparseable.

HELPER: `_parse_duration(s: str) -> timedelta | None`
Parse "Xm", "Xh", "Xd" (case-insensitive). Return None if invalid.

HELPER: `_next_weekday_at(day_name: str, time_str: str) -> datetime`
Compute the next UTC datetime for the given weekday name ("Monday".."Sunday") at time_str ("09:00" 24h format). If today is that weekday but the time has already passed, return next week. Always returns timezone-aware UTC datetime.

HELPER: `_has_manage_guild(interaction: discord.Interaction) -> bool`
Return `interaction.user.guild_permissions.manage_guild` (member only, not webhooks).

BACKGROUND TASKS:

`async def _run_close_at(self, poll_id: int, close_at: datetime) -> None`
Sleep until close_at (compute delta from now, clamp to 0 if past). Call `await self._close_poll(poll_id)`.

`async def _run_post_then_close_task(self, poll_id: int, post_at: datetime, close_at: datetime) -> None`
Sleep until post_at. Post the poll embed and add reactions. Sleep until close_at. Call `await self._close_poll(poll_id)`.

`async def _run_weekly_post_task(self, recurring_id: int) -> None`
Loop indefinitely:
1. Fetch recurring_poll row; if active=0 → break
2. Compute next post_at and close_at from day_of_week, post_time, close_time
3. If now is past the close_at of the current week's slot (missed while offline), post immediately then schedule close immediately
4. Else sleep until post_at
5. Create a new poll instance row (INSERT_POLL with post_at=None, recurring_poll_id=recurring_id)
6. Build embed, post, add reactions, UPDATE_POLL_MESSAGE_ID
7. Sleep until close_at
8. `await self._close_poll(new_poll_id)`
9. Loop again for next week

`async def _close_poll(self, poll_id: int) -> None`
1. Fetch poll via SELECT_POLL_BY_ID; return if already closed
2. CLOSE_POLL (mark closed=1)
3. Fetch votes and voters
4. Build results summary embed (see format)
5. Edit the original message (fetch via channel.fetch_message(int(discord_message_id))) and send a follow-up results message to the same channel
6. If poll has recurring_poll_id and the recurring poll is still active: log "next instance will be posted by weekly task"

REACTION HANDLER: `on_raw_reaction_add`

```
1. Ignore bot reactions
2. Check if reaction emoji is in _EMOJI_TO_IDX — ignore all others
3. Fetch message channel; look up poll by discord_message_id (SELECT_POLL_BY_MESSAGE_ID)
4. If not found or poll.closed → remove reaction and return
5. option_idx = _EMOJI_TO_IDX[emoji_name]
6. Validate option_idx < len(options)
7. Fetch user's current vote count (SELECT_VOTE_COUNT_BY_USER)
8. max_choices = poll["max_choices"] (None = unlimited)

If max_choices is None (unlimited):
  - INSERT_POLL_VOTE (INSERT OR IGNORE)
  - return

If max_choices == 1 (radio button):
  - Fetch current votes for this user (SELECT_VOTES_BY_USER)
  - If already voted for this option → return (no-op)
  - If voted for a different option:
      - DELETE_POLL_VOTE for the old option
      - Remove old reaction from message via message.remove_reaction(old_emoji, user)
  - INSERT_POLL_VOTE

If max_choices > 1:
  - current_vote_count = SELECT_VOTE_COUNT_BY_USER result
  - If current_vote_count >= max_choices:
      - Remove the new reaction via message.remove_reaction(emoji, user)
      - Send ephemeral DM-style: try interaction... but we're in a raw event, so
        use channel.send with a mention then delete after 10s:
        await channel.send(
            f"<@{payload.user_id}> You can only pick {max_choices} option(s) in this poll. "
            "Remove one of your current votes first.",
            delete_after=10,
        )
      - return
  - INSERT_POLL_VOTE
```

For removing reactions, fetch the message: `channel = self.bot.get_channel(payload.channel_id)`, then `msg = await channel.fetch_message(payload.message_id)`, then `await msg.remove_reaction(payload.emoji, discord.Object(id=payload.user_id))`.
